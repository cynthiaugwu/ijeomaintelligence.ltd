About Ijeoma

Ijeoma Intelligence is a real-time safety intelligence platform designed for organizations and individuals working in high-risk environments. By computing risk per person, route, and moment, Ijeoma combines live threat signals, environmental exposure, and behavioral patterns to continuously assess current safety levels. Built on the principle of proactive safety intelligence, the platform assumes users are unsafe until proven otherwise, ensuring continuous safety verification. Initially launching in Nigeria, Ijeoma aims to serve the world's most complex and challenging operating environments.

The Role

We need an engineer to own the data infrastructure and backend systems that power the Ijeoma platform. This includes the data pipeline, backend architecture, enterprise web platform, and admin tooling. You will work directly with the founder, who will provide detailed technical specifications under contract.

What You'll Work On

  • Design and maintain the backend data architecture across multiple data sources
  • Build and harden data ingestion pipelines with classification, validation, and deduplication
  • Develop the enterprise web platform for organisational clients — real-time monitoring, team oversight, compliance reporting
  • Extend the existing admin dashboard with operational management tools
  • Build backend services that synthesise multiple data streams into actionable intelligence
  • Define and maintain API contracts consumed by the mobile application
  • Ensure structured event capture across all system touchpoints with proper schema design

Full technical specifications and system architecture will be shared under NDA.

Requirements

  • 5+ years backend engineering with production systems
  • Strong database design skills — Firestore, PostgreSQL, or equivalent
  • Experience building data pipelines and ingestion systems
  • Advanced Python proficiency
  • React or Next.js for web platform development
  • Experience building B2B SaaS platforms or enterprise dashboards
  • Comfortable with geospatial data and real-time systems
  • Security-conscious — API hardening, role-based access, data protection
  • Self-directed and able to make sound architectural decisions independently

Nice to Have

  • At least 1 enterprise platform built and deployed
  • Experience with safety, security, intelligence, or logistics platforms
  • Web scraping and data quality systems
  • Building for emerging market infrastructure constraints
